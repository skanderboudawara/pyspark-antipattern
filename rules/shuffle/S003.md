# Rule S003  
`.groupBy()` directly followed by `.distinct()`

## Information  
Using `.groupBy()` followed by `.distinct()` is redundant because both operations trigger data shuffling and aggregation. This can lead to:

- Unnecessary extra computation  
- Increased shuffle overhead  
- Slower performance  

## Best practices  
- Remove `.distinct()` if `.groupBy()` already ensures uniqueness  
- Use only `.distinct()` when no aggregation is needed  
- Ensure you are not duplicating operations that achieve the same result  

**Rule of thumb:** Avoid chaining `.groupBy()` and `.distinct()`—use only the operation that fits your goal.
